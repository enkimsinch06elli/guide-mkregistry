# guide-mkregistry

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存放，不针对任何特定域名或具体网站，仅作为页面资源的统一存储与版本管理。

## 目录说明

```
guide-mkregistry/
├── pages/            # 存放所有独立 HTML 页面
├── assets/           # 页面引用的静态资源（如图片、样式、脚本）
└── README.md         # 本文件
```

- `pages/`：每个 HTML 文件独立存在，可包含各自样式与脚本。
- `assets/`：公共资源建议放在此处，方便多个页面共享。

## 页面归档说明

- 所有页面均以 `.html` 后缀命名，文件名简洁明了。
- 页面内容不包含任何外部链接或跳转行为，仅呈现静态信息。
- 如需新增页面，请直接放入 `pages/` 目录，并确保引用资源路径正确。

## 维护说明

- 本仓库主要用于归档，不建议频繁修改已有页面内容。
- 若需更新，请遵循以下原则：
  - 保持原有文件路径与命名不变。
  - 修改后请更新本文件中的相关说明（如有必要）。
- 欢迎通过 Issue 提交页面建议或问题反馈，但请勿添加具体推广或营销内容。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/guide-mkregistry.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看。
3. 也可将仓库部署至静态托管服务，实现页面在线访问（不依赖任何特定域名）。

## 贡献

- 提交 Pull Request 前请确保页面符合归档规范。
- 避免引入外部依赖或动态脚本，以保持页面轻量与独立。
